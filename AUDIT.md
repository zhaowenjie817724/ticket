# 极速抢审计报告

## 审计范围

- `index.html`
- `styles.css`
- `app.js`
- `src/strategy.js`
- `sw.js`
- `tests/strategy.test.mjs`
- 发布配置和文档

## 结论

当前实现是合规购票作战工具，不是自动抢票脚本。项目只做本地配置、倒计时、冲刺模式、官方入口跳转、PWA 离线缓存和用户侧性能专注辅助。

## 已验证

- 策略单元测试通过：`npm test`
- JavaScript 语法检查通过：`node --check app.js`、`node --check src/strategy.js`
- URL 白名单测试覆盖 12306 和大麦域名。
- 大麦项目 ID 可以生成网页入口和移动入口。
- 12306 可在填写电报码后生成官方余票页参数链接。
- 冲刺时间按目标开售时间提前 N 分钟计算。

## 合规边界

本项目没有实现以下能力：

- 自动提交订单。
- 自动锁票。
- 验证码识别。
- 打码平台接入。
- 私有接口调用。
- 代理池。
- 多账号并发。
- Selenium/Appium 自动点击。
- Cookie 窃取或账号凭证上传。

## 数据安全

- 所有配置保存在用户浏览器 `localStorage`。
- 没有后端服务器。
- 没有上传账号、手机号、实名信息或支付信息。
- 没有第三方统计脚本。

## 风险和限制

- 手机 App 深链由系统和官方 App 决定是否接管；如果不接管，会回落到官方网页。
- 浏览器可能限制自动打开新窗口；本项目采用当前页面跳转来提高稳定性。
- Wake Lock、Fullscreen、Notification 需要浏览器支持和用户授权。
- Web App 不能真正接管操作系统网络优先级，只能通过关闭干扰、保持屏幕、预热连接和网络自检提高稳定性。

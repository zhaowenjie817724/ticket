## 可吸收能力

- 配置化目标：把出发地/目的地/票档/项目 ID/目标链接作为结构化配置。
- 倒计时状态机：区分“未启动”“冲刺窗口”“正式开抢”。
- 多入口策略：网页入口和移动入口同时准备。
- 手机入口兜底：App 深链、Android intent、官方移动网页、官方电脑网页按顺序准备。
- 链接解析：从大麦详情页链接提取 `itemId`，减少用户手动填写成本。
- 前置校验：账号、实名、支付、目标信息、网络状态在开抢前完成。
- 快速回退：首选目标失败后快速切换备选票档、席别、候补或回流窗口，但只给用户提供简洁入口，不展示冗长执行清单。

## 明确排除能力

- 不做自动提交订单。
- 不做自动锁票。
- 不做验证码识别或打码平台接入。
- 不做接口逆向、抓包下单或私有 API 调用。
- 不做代理池、多账号并发或刷票。
- 不做 Appium、Selenium 自动点击官方页面。

## 本项目落地

- 新增“抢票冲刺模式”：用户手动启动后进入性能专注状态。
- 支持开抢前 N 分钟自动跳转官方入口，默认 5 分钟。
- 支持电脑网页入口和手机 App/移动入口分流。
- 手机端按钮会按候选入口队列尝试：自定义入口、App 深链、Android intent、官方移动页、官方网页。
- 顾客端 UI 去掉“执行梯队”和“开抢前检查”，把复杂策略内化为准备度、倒计时和入口状态。
- 支持 Wake Lock 防息屏、全屏专注、网络状态检查、DNS/连接预热。
- 只跳转官方入口，最终提交订单仍由用户本人完成。
